Разбор полей:

    "crystal": Уникальный ключ (любое слово на латинице, главное чтобы не повторялось).

    "name": Как программа будет называться в меню WPI (на экране).

    "cat": Категория (должна совпадать с одной из тех, что в SortOrder в шапке скрипта).

    "uid": Уникальный ID для WPI (любое слово капсом, например CDI).

Нюансы (Pro Tips)
1. Как определяется x64 или x86?

Скрипт определяет разрядность автоматически по имени файла.

    Если в имени файла есть: x64, win64, 64bit -> он поставит {x64}.

    Если есть x86, win32 -> он поставит {x86}.

    Если ничего нет -> он по умолчанию поставит {x86} (как в твоем логе с Firefox и Epic Games).

Совет: Если ты качаешь 64-битную версию, но в названии файла нет "x64" (например FirefoxSetup.exe), лучше переименовать его при скачивании в Jenkinsfile (добавить ?name=Firefox_x64.exe), чтобы WPI правильно понял архитектуру.
